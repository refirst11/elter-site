## Integration
you need to configure it in your configuration file.
<Tabs items={['Next.js', 'Vite', 'Webpack']}>
  {/* prettier-ignore */}
  <Tab>
    ```js title="next.config.js"
    const { TypedCSSXNextPlugin, configCSSModule } = require('typedcssx-next-plugin')

    /** @type {import('next').NextConfig} */
    const nextConfig = {
      webpack: (config) => {
        config.plugins.push(new TypedCSSXNextPlugin())
        return configCSSModule(config)
      },
    }

    module.exports = nextConfig
    ```
  </Tab>
  {/* prettier-ignore */}
  <Tab>
    ```js title="vite.config.js"
    import { defineConfig } from "vite";
    import { nodePolyfills } from "vite-plugin-node-polyfills";

    export default defineConfig({
      plugins: [nodePolyfills({ include: ["fs", "path"] })],
    });
    ```
  </Tab>
  {/* prettier-ignore */}
  <Tab>
    ```js title="webpack.config.js"
    const { TypedCSSXWebpackPlugin } = require("typedcssx-webpack-plugin");

    module.exports = {
      plugins: [new TypedCSSXWebpackPlugin()]
    };
    ```
  </Tab>
</Tabs>

## Compile Command

Add to your build command <code>\`npx typedcssx --compile\`</code>

<Tabs>
  <Tab>
  ```json title="package.json"
    "scripts": {
      "build": "npx typedcssx --compile && next build",
    },
  ```
  </Tab>
</Tabs>

## Server Component
You can statically preview a ServerComponent using `ServerStylePreview`.
```tsx title="layout.tsx"
import { ServerStylePreview } from 'typedcssx/next'

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html>
      <head>
        <ServerStylePreview />
      </head>
      .....
  )
}    
```

## Supported Node.js

This project has been tested and is confirmed to work with Node.js versions from 18 ~ ^20.  
Please you are using one of these versions for optimal compatibility and performance.  
@types/node version is ^22.5.4. and typescript version is ^5.4.5.
