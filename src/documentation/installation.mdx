## Compiler setting

Add the compile command to the `package.json` script field.

<Tabs>
  {/* prettier-ignore */}
  <Tab>
  ```json title="package.json"
    "scripts": {
      "build": "npm run compile && next build",
      "compile": "cd node_modules/typedcssx && npm run compiler"
    },
  ```
  </Tab>
</Tabs>

Add css.ts and css.tsx to icludes in tsconfig.json.
<Tabs>
  <Tab>
  ```json title="tsconfig.json"
    "include": ["**/*css.ts", "**/*css.tsx"],
  ```
  </Tab>
</Tabs>

## Server Component Preview
If the anchorEnabled argument is false, you will not be able to see the server component styles unless you manually reload the page once, but if it is true, you can collect the server component styles by mimicking the behavior of the a tag and reloading the page when transitioning via Link.
This true behavior remembers the page you clicked once and will not reload it thereafter.
```tsx title="layout.tsx"
import { PreviewServerCSS } from 'typedcssx'

    <html>
      <head>
        <PreviewServerCSS anchorEnabled={true} />
      </head>
      <body>
          {children}
      </body>
    </html>
```

## Global style sheet
Load global.css in layout.tsx, which is the root directory of the app.
<Tabs>
  <Tab>
```tsx title="layout.tsx"
import '../../node_modules/typedcssx/dist/core/styles/global.css'
```
  </Tab>
</Tabs>

## Supported Node.js

This project has been tested and is confirmed to work with Node.js versions from v18 to v22.  
Please you are using one of these versions for optimal compatibility and performance.
